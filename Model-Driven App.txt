# Power Apps Model-Driven Apps - Complete Interview Notes

# 1. What is a Model-Driven App?

A Model-Driven App is a type of Power App where the application's UI is automatically generated based on the Dataverse data model.

Interview Answer:

A Model-Driven App is a data-driven application built on Microsoft Dataverse where the user interface is automatically generated based on tables, relationships, forms, views, and business processes.

---

# 2. Why Use Model-Driven Apps?

Advantages:

* Faster development
* Minimal UI design effort
* Responsive by default
* Built-in security
* Supports complex business processes
* Rich Dataverse integration

---

# 3. Architecture of Model-Driven Apps

User
↓
Model-Driven App
↓
Dataverse
↓
Tables + Forms + Views + Business Logic

Everything in Model-Driven Apps revolves around Dataverse.

# 4. Main Components of Model-Driven Apps

Tables

Tables store business data.

Examples:

* Account
* Contact
* Employee
* Service Request

Tables are mandatory in Model-Driven Apps.

---

Columns

Columns store data within tables.

Examples:

* Name
* Status
* Department
* Amount

Common Column Types:

* Text
* Number
* Choice
* Lookup
* Date and Time
* Currency
* Yes/No

# Relationships

Types:

One-to-Many (1:N)

Many-to-One (N:1)

Many-to-Many (N:N)

#  Canvas App vs Model-Driven App

| Feature     | Canvas App           | Model-Driven App    |
| ----------- | -------------------- | ------------------- |
| UI Design   | Fully customizable   | Auto-generated      |
| Data Source | Multiple sources     | Primarily Dataverse |
| Development | UI-first             | Data-first          |
| Responsive  | Manual               | Automatic           |
| Complexity  | Simple to Medium     | Medium to Complex   |
| Best For    | Custom business apps | Enterprise CRM apps |

Interview Question:

Q: When would you choose a Model-Driven App over a Canvas App?

Answer:
I would choose a Model-Driven App when the application requires complex relational data, role-based security, standardized business processes, and is heavily dependent on Dataverse.
