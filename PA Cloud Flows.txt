# Power Automate Cloud Flows - Complete Interview Notes

# 1. What is Power Automate?

Power Automate is a low-code automation platform used to automate business processes, integrate applications, and reduce manual work.

Interview Answer:

Power Automate is a cloud-based service that enables users to automate repetitive business processes across multiple applications and services using triggers and actions.

---

# 2. What is a Cloud Flow?

A Cloud Flow is an automated workflow that runs in the cloud and performs actions based on triggers.

Example:

Employee submits leave request
↓
Manager receives approval email
↓
Manager approves
↓
Employee receives notification

Interview Answer:

Cloud Flows are automated workflows that run in Microsoft's cloud environment and automate business processes by connecting various applications and services.

---

# 3. Types of Cloud Flows

## A. Automated Cloud Flow

Runs automatically when an event occurs.

Examples:

* When a SharePoint item is created
* When an email arrives
* When a file is uploaded

Example:

New SharePoint Item Created
↓
Send Email Notification

Interview Question:

Q: What is an Automated Cloud Flow?

Answer:
An Automated Cloud Flow starts automatically when a specified event occurs.

---

## B. Instant Cloud Flow

Runs manually by a user.

Triggers:

* Button click
* Power Apps trigger
* Manual trigger

Examples:

* User clicks Submit button in Power Apps.
* User manually starts a process.

Interview Question:

Q: What is an Instant Cloud Flow?

Answer:
An Instant Cloud Flow runs on-demand when initiated manually by a user or another application such as Power Apps.

---

## C. Scheduled Cloud Flow

Runs at predefined intervals.

Examples:

* Daily reports
* Weekly reminders
* Monthly cleanup jobs

Example:

Every day at 8 AM
↓
Generate Report
↓
Send Email

Interview Question:

Q: What is a Scheduled Cloud Flow?

Answer:
A Scheduled Cloud Flow executes automatically based on a defined schedule.

---

# 4. Components of a Cloud Flow

Every flow consists of:

## Trigger

The event that starts the flow.

Examples:

* When an item is created
* Power Apps trigger
* Recurrence trigger

## Actions

Tasks performed after the trigger.

Examples:

* Create Item
* Send Email
* Update Record

Flow Structure:

Trigger
↓
Action 1
↓
Action 2
↓
Action 3

# 13. Scopes

Scopes group actions logically.

Common Pattern:

### Try Scope

Main processing

### Catch Scope

Error handling

### Finally Scope

Cleanup activities

Interview Question:

Q: Why do you use Scopes?

---

# 14. Error Handling (Very Important)

Techniques:

## Configure Run After

Can execute actions after:

* Success
* Failure
* Timeout
* Skipped

Example:

Try Scope
↓
Catch Scope (runs if Try fails)

---

## Exception Handling Pattern

Try
↓
Catch
↓
Finally

Interview Question:

Q: How do you handle errors in Power Automate?

Answer:
I use Scope actions combined with Configure Run After settings to implement Try-Catch-Finally error handling. I also log errors and send failure notifications.

# 17. Child Flows

A Child Flow is a reusable flow called by another flow.

Benefits:

* Reusability
* Easier maintenance

Interview Question:

Q: What are Child Flows?

Answer:
Child Flows are reusable flows that can be invoked from parent flows to modularize business logic.
