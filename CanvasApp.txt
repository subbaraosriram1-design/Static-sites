# Power Apps Canvas Apps - Complete Interview Notes

# 1. What is a Canvas App?

A Canvas App is a low-code application where developers have full control over the User Interface (UI) using drag-and-drop controls and Power Fx formulas.

Interview Answer:

A Canvas App is a type of Power App that provides complete control over the application's user interface. It allows developers to design screens, place controls anywhere on the canvas, and connect to multiple data sources using Power Fx formulas.

---

# 2. Why Canvas Apps?

Advantages:

* Full UI customization
* Supports responsive design
* Connects to multiple data sources
* Low-code development
* Mobile and web support
* Fast application development

---

# 4. Components of Canvas Apps

## A. Screens

Screens are pages within the application.

Examples:

* Home Screen
* Dashboard Screen
* Create Screen
* Edit Screen
* Approval Screen

Important Functions:

```PowerFx id="h2f7r1"

```

Interview Question:

Q: What are Screens?

Answer:
Screens are containers that hold controls and represent different pages in a Canvas Application.

# 7. Variables

## Global Variables

Accessible throughout the app.

Syntax:

```PowerFx id="g6p3x1"
Set(varUser, User().FullName)
```

## Context Variables

Accessible only within a screen.

Syntax:

```PowerFx id="v9q2n4"
UpdateContext({showPopup:true})
```

## Collections

Temporary in-memory tables.

Syntax:

```PowerFx id="m8r5t7"
ClearCollect(
    colEmployees,
    EmployeeList
)
```

# 9. CRUD Operations

## Create

```PowerFx id="k3x9b2"
Patch(
    EmployeeList,
    Defaults(EmployeeList),
    {
        Name: txtName.Text
    }
)
```

## Read

```PowerFx id="u8c4d7"
Filter(EmployeeList, Status="Active")
```

## Update

```PowerFx id="e6h2q5"
Patch(
    EmployeeList,
    Gallery1.Selected,
    {
        Status:"Approved"
    }
)
```

## Delete

```PowerFx id="w4m8p1"
Remove(EmployeeList, Gallery1.Selected)
```

# 10. Patch Function (Very Important)

Patch is used to create or update records.

Interview Question:

Q: Why did you use Patch instead of SubmitForm?

Answer:
Patch provides greater flexibility for updating specific fields, working with multiple data sources, and implementing custom business logic.

---

# 11. Delegation (Most Important)

Delegation means processing data at the data source instead of locally.

Benefits:

* Better performance
* Handles large datasets

Common Delegable Functions:

* Filter
* LookUp
* Sort
* StartsWith

Interview Question:

Q: What is Delegation?

Answer:
Delegation is the process of sending queries to the data source for processing rather than retrieving all records to the application.

# 16. Performance Optimization (Very Important)

Techniques:

* Use delegable queries.
* Minimize collections.
* Avoid nested galleries.
* Use Concurrent().
* Load only required columns.
* Avoid unnecessary Refresh().
* Use indexed SharePoint columns.
* Use Monitor tool.
