# Power Pages - Complete Interview Notes

# 1. What is Power Pages?

Power Pages is a low-code platform used to create secure, external-facing websites that allow users outside the organization to interact with business data stored in Dataverse.

Examples:

* Customer Portal
* Vendor Portal
* Citizen Services Portal
* Employee Self-Service Portal

Interview Answer:

Power Pages is a low-code SaaS platform in Microsoft Power Platform that enables organizations to build secure, external-facing business websites integrated with Dataverse.

---

# 2. Why Use Power Pages?

Advantages:

* Build websites quickly.
* External users can access data.
* Secure authentication and authorization.
* Seamless integration with Dataverse.
* Responsive design.
* Integration with Power Automate and Power BI.

---

# 3. Typical Architecture

External User
↓
Power Pages Website
↓

↓

# 4. Main Components of Power Pages

## A. Website

The complete portal application.

Examples:

* Customer Portal
* Supplier Portal

A website contains:

* Web Pages
* Navigation
* Forms
* Lists
* Authentication

---

## B. Web Pages

Web Pages represent pages within the portal.

Examples:

* Home Page
* Contact Us
* Product Page
* Dashboard Page

Properties:

* Name
* URL
* Parent Page
* Permissions

---

## C. Page Templates

Templates define the layout and structure of pages.

Examples:

* Home Template
* Landing Page Template

---

## D. Web Templates

Reusable HTML/Liquid templates.

Used for:

* Dynamic page rendering
* Custom layouts

Technologies Used:

* HTML
* CSS
* JavaScript
* Liquid

Interview Question:

Q: What is a Web Template?

Answer:
A Web Template is a reusable HTML and Liquid-based template used to define page layouts and dynamically render content in Power Pages.

# 9. Authorization (Very Important)

Authorization controls what users can access.

Power Pages uses:

## Web Roles

Examples:

* Customer
* Vendor
* Administrator

## Table Permissions

Control record-level access.

Permissions:

* Create
* Read
* Write
* Delete
* Append
* Append To

Interview Question:

Q: How is security implemented in Power Pages?

Answer:
Security in Power Pages is implemented using Web Roles and Table Permissions, which control access to Dataverse records.

---

# 10. Web Roles

Web Roles define portal user permissions.

Examples:

* Anonymous Users
* Authenticated Users
* Customer Users
* Admin Users

Example:

Customers can view only their own requests.

---

# 11. Table Permissions

Controls access to Dataverse tables.

Scopes:

### Global

Access to all records.

### Contact

Access only to records related to logged-in user.

### Account

Access to records related to user's account.

### Self

Access only to own records.

Interview Question:

Q: What are Table Permissions?

Answer:
Table Permissions define CRUD access to Dataverse records within Power Pages.

# 15. Canvas Apps vs Power Pages

| Feature        | Canvas Apps         | Power Pages         |
| -------------- | ------------------- | ------------------- |
| Users          | Internal users      | External users      |
| Access         | Employees           | Customers/Vendors   |
| Authentication | Microsoft 365 users | External identities |
| Data Source    | Multiple            | Primarily Dataverse |
| Deployment     | Mobile/Web App      | Website             |

Interview Question:

Q: When would you choose Power Pages?

Answer:
Power Pages is preferred when external users such as customers, vendors, or partners need secure access to business data through a web portal.
